1. På Wikipedia står det: (Wiki-länk) ”JavaScript often abbreviated as JS, is a high-level, interpreted programming language.”
Vad innebär denna definition, förklara?
Tips: JavaScript: Interpreted or Compiled?

Svar: När man pratar om high-level språk så innebär det att man talar om vad man ska göra inte hur. Motsatsen är low-level språk och då är man nere på maskinkodning eller assembling som det heter (där man berättar hur det ska utföras av datorns processor).För att ta ett exempel kan man säga att high-level är som att gå på restaurang, man säger vad man vill äta så får man maten serverad, medans low-level är att gå in och laga maten själv. Dvs man talar om hur man ska hacka grönsakerna, hur många grader ungen ska vara på eller hur mycket salt och peppar man ska ha.

I low-level språk går man in och detaljstyr i större utsträckning än i high-level språk. Man säger att high-levelspråk har en hög abstraktionsnivå. Det innebär att man har "tagit bort" svårbegripliga binärfunktionerna i tex maskinkod och landat i ett språk som enklare kan beskrivas i ord och enklare instruktioner.

Javascript är ett interpreterat språk, i och med att det kräver en programtolk för att köras (JavaScriptmotor i webbläsaren eller node.js). Dock kan beteendet upplevas som ett kompilerat språk i och med att man exempelvis blir medveten om eventuella buggar innan man kör scriptet.

2. JavaScript använder ”dynamisk typning” (dynamic typing) Vad innebär det? Förklara och skriv några exempel. Tips: Dynamic typing

Svar: I ett dynamiskt språk som i det här fallet, Javascript, så kan variabler ändra värde. Alltså att man kan tilldela tex variabel x först ett nummer och sedan tilldela den ett nytt värde tex string. Då kommer det få det senaste värdet som man har tilldelat x. Javascript är ett språk där variablerna inte har ett värde av en specifik datatyp (ex Number, Boolean, String, Undefined, Null) utan kan anta alla olika datatyper utan att Javascript klagar.

3. Vad är console.log() och vad kan det användas till? Beskriv två andra användbara metoder som finns i console-objektet Tips: The Console Object

Svar: Console är ungefär som en tolk som hjälper programmeraren att se vad som händer när man kör kod, så att allt fungerar som det var tänkt. Console är där den exekverade koden visas, till exempel går det att använda webbläsaren som en console/tolk (i Chrome kan man trycka ALT + CMD + j). För Mac-användare heter tolken Terminal, för windows kallas den vanligaste Bash eller Powershell. Man kan även ladda ned plugins som till exempel Hyper.

Metoden som gör att man kan se kod på Console/Kommandotolken heter log(). Man skriver därför console.log(). Det man vill att console ska skriva ut står mellan parenteserna.

Till exempel kan man skriva;
number - console.log(1234)
string - console.log("a,b,c")
eller deklarerade variabler - console.log(firstName)

4. För variabler används begreppen deklarera (declare) och tilldela (assign). Vad innebär dessa?

Svar: Att deklarera (declare) en variabel innebär att man namnger varibeln som man senare kan spara olika värden i. När man har gjort det kan man tilldela (assign) variabeln ett värde.

Deklaration (declare):
let x;
Tilldela värdet 5 i variabeln x (assign):
x = 5;
Skriver ut värdet i tolken;
console.log(x);

Att deklarera och tilldela värde i samma rad:
let x = 5;

    7. Vad är NaN?

    Svar: NaN är förkortning för Not a Number, ett värde som indikerar att värder ej är ett nummer. Returneras som värde när matematiska funktioner misslyckas.
    Exempelvis, det returnerade värdet på 0/0 i JavaScript är NaN.

    8. Vad används semikolon till? Är det obligatoriskt att använda ; i JavaScript?

    Semikolon används i JavaScript för att separera kommandon. Det är inte obligatoriskt med semikolon i JavaScript, men är det nödvändigt om det körs två kommandon på samma rad. Ex:
    let x = 0; x++
    Dock lägger JavaScriptmotorn in semikolon vid exempelvis radbrytningar, vilket gör att följande kod fungerar utan semikolon:
    let x = 0
    x++

    Även fast semikolon inte är obligatoriskt att använda överallt (förutom på vissa platser) är det ansett att vara best practise att ändå använda semikolon för att separera varje kommando. Största argumentet för detta är att själv ha kontroll över vart dessa sätts, för att säkerställa att JavaScript-motorn inte infogar dessa på felaktiga platser som kan resultera i att koden beter sig annorlunda/felaktigt.

    9. Vad är aritmetiska operatorer och hur kan de användas tillsammans med variabler?

    Aritmetiska operatorer är matematiska verktyg.
    De som finns i JavaScript är: addition (+), subtraktion (-), multiplikation (*), division (/). Subtraktion, multiplikation, division används på samma sätt som i matematiken.
    Det finns några undantag: increment (++), decrement (--), modulus (%).
    Sedan finns det ännu ett undantag med operator + som kan anvädas som kompilator, dvs sätta ihop exempelvis strängar, variabler, funktioner, arrayer och objekt.
    Exempel: let a = b + c; Om vi skulle skriva: console.log(a) skulle "bc" skrivas ut i consolen.

    10. Undersök följande kodsnutt.
    let cars = new Array(2);
    cars[cars.length] = "Volvo";
    Hur många element rymmer arrayen cars?

    Array(2) skapar en array med två tomma platser. Inte en array med index 0 innehållande nummer 2. När man då lägger till "Volvo" på index 2 (som definieras av cars.length = 2) blir det 3 element, för att den adderar "Volvo" på den platsen.

    Vad innehåller arrayen exakt?

    Den innehåller två tomma platser och sen "Volvo", dvs: [Empty, Empty, "Volvo"]
